## 1. Концепция

Opinion Market — платформа для слепых ставок на вопросы-мнения. Пользователи выбирают сторону и делают ставку, не видя текущих объёмов. Побеждает сторона, на которую поставили больше денег. После закрытия маркета объёмы раскрываются, победители делят банк пропорционально своим ставкам.

Это не prediction market в классическом смысле — здесь нет "правильного" ответа и внешнего оракула. Исход определяется коллективным мнением, выраженным в деньгах. Ты ставишь не на то, что произойдёт, а на то, что думает большинство.

### Почему это работает

**Blind механика** — объёмы скрыты до конца. Нет возможности "оседлать тренд" или скопировать сторону большинства. Каждая ставка — выражение собственного мнения, а не реакция на чужие.

**Одна транзакция** — поставил и забыл. ZK proof скрывает сторону и сумму on-chain. Результат приходит автоматически.

**User-generated** — любой пользователь может создать маркет на любую тему. Лучшие маркеты — виральные, провокационные вопросы, которые хочется расшарить.

**On-chain, trustless** — ZK proof гарантирует, что объёмы действительно скрыты, а не просто спрятаны сервером. Контракт верифицирует каждую ставку. Выплаты автоматические.

### Пример

> "Кто победит на выборах мэра — Кандидат А или Кандидат Б?"

Маркет открыт 16 часов. 200 человек поставили на A, 150 на B. Но объём на B оказался выше: $12,000 против $8,000 на A. Побеждает B. Бетторы B делят весь банк ($20,000 минус 4% комиссия) пропорционально своим ставкам.

## 2. Механика V0

### Маркет

Маркет = вопрос с двумя вариантами ответа. В V0 один раунд: ставки принимаются в течение фиксированного периода (8 или 16 часов), после чего маркет закрывается.

Жизненный цикл:
```
CREATED → ACTIVE (ставки принимаются) → RESOLVED (подсчёт) → SETTLED (выплаты)
```

### Ставки (ZK Deposit)

Пользователь делает ставку одной транзакцией. ZK proof скрывает сторону (A или B) и сумму.

1. Пользователь выбирает сторону + сумму в UI
2. Клиент генерирует ZK proof: `prove(side, amount, salt)` → `(commitment, proof)`
3. Транзакция: `placeBet(commitment, proof)` + депозит USDC
4. Контракт верифицирует proof и сохраняет commitment
5. Средства заблокированы в контракте

**Что скрыто:** сторона (A/B), точная сумма
**Что видно:** количество бетторов, факт ставки


### Отсечка и резолюция

По истечении таймера:
1. Контракт закрывает приём ставок
2. Все commitments раскрываются через ZK verification
3. Подсчёт объёмов: `total_A` vs `total_B`
4. Победитель = сторона с большим объёмом
5. При равенстве — ничья, ставки возвращаются

### Выплаты

```
total_pool = total_A + total_B
fee = total_pool * 0.04                    // 4% комиссия
payout_pool = total_pool - fee

// Каждый победитель:
user_payout = (user_bet / total_winning_side) * payout_pool

// Fee distribution:
platform_share = fee * 0.50                // 50% — платформа
creator_share  = fee * 0.25               // 25% — создатель маркета (~1% от выплат)
referral_pool  = fee * 0.25               // 25% — рефералы
```

### Лимиты

- Минимальная ставка: $1
- Максимальная ставка: без ограничений
- Минимальный пул для запуска маркета: $30 (иначе маркет отменяется, ставки возвращаются)

## 3. Scope V0

Фичи, которые входят в первый тестовый запуск:

- Один раунд на маркет (8 или 16 часов)
- ZK deposit proof (скрытие стороны и суммы)
- User-generated маркеты с creation fee ($5 USDC, возвращается при достижении min_pool)
- Seed-ставка от создателя (опционально)
- 5 категорий: Крипто, Политика, Спорт, Культура/мемы, Другое
- Базовая модерация: автофильтр + community report
- Реферальная система (1 уровень, % от fee)
- Показ количества бетторов (без объёмов) во время активного маркета
- Подсказки для создателей: примеры memeable формулировок вопросов
- Monad L2, USDC
- 4% комиссия (50% платформа / 25% создатель / 25% рефералы)

- Chainlink VRF для рандомных отсечек
- Награды (лучший маркет недели, бонусы создателям)

**Не входит в V0:**
- Мультиблок (3-5 блоков, 30%/70% перенос)
- Многоуровневая реферальная система
- Fiat on-ramp
- Расширенная аналитика

## 4. User-generated маркеты

### Создание маркета

Любой пользователь может создать маркет:
1. Написать вопрос (макс. 280 символов)
2. Задать два варианта ответа
3. Выбрать категорию
4. Выбрать длительность (8h / 16h)
5. Заплатить creation fee ($5 USDC) — возвращается если маркет набрал min_pool
6. Опционально: сделать seed-ставку (skin in the game)

В UI показываем подсказки и примеры хороших формулировок, чтобы маркеты были виральными и провокационными. Скучные вопросы не набирают ставок.

### Модерация (V0 — минимальная)

- Автофильтр: запрещённые слова, спам-детекция
- Community report: пользователи жалуются → маркет скрывается после N жалоб
- Нет ручной модерации в V0

## 5. Реферальная система

### V0 — простая модель

Реферер получает 25% от реферального пула (25% от fee) с каждой ставки своего реферала. Реферальный доход берётся из доли платформы, а не из выигрыша бетторов — нет конфликта интересов.

Механика:
- Каждый пользователь получает уникальный реферальный код (on-chain)
- Код привязывается при первой ставке реферала
- Привязка постоянная (первый реферер навсегда)

Anti-abuse:
- Минимальный депозит $10 для активации реферального кода
- Cap на реферальный доход с одного реферала: $50/маркет

### V1 — трёхуровневая система (после валидации V0)

Многоуровневые рефералы значительно усиливают виральный loop (подтверждено Belief Market):

- Уровень 1 (прямой реферал): 50% от tx fees
- Уровень 2 (реферал реферала): 20% от tx fees
- Уровень 3: 5% от tx fees

Плюс tiered loyalty: чем больше привёл, тем выше процент. Бонусы для рефералов (сниженная комиссия).

## 6. User Flows

### Flow 1: Первая ставка

1. Пользователь попадает на Home по реферальной ссылке
2. Видит список активных маркетов: вопрос, таймер, количество бетторов
3. Выбирает интересный маркет → переходит на Market page
4. Видит вопрос, две стороны (A/B), таймер обратного отсчёта, количество бетторов (объёмы скрыты)
5. Подключает кошелёк (WalletConnect)
6. Выбирает сторону, вводит сумму ($1 минимум)
7. Нажимает "Поставить" → клиент генерирует ZK proof → одна транзакция
8. Видит подтверждение: "Ставка принята. Результат — через Xч Yмин"
9. Получает push-уведомление при закрытии маркета
10. Переходит на Result page: видит объёмы A vs B, победителя, свою выплату
11. Нажимает "Claim" → получает выигрыш на кошелёк

### Flow 2: Создание маркета

1. Пользователь нажимает "Создать маркет" на Home
2. Заполняет форму: вопрос, два варианта, категория, длительность
3. UI показывает подсказки: "Используй провокационные формулировки", примеры хороших маркетов
4. Видит creation fee: $5 USDC (возвращается при наборе $30 в пуле)
5. Опционально: делает seed-ставку на одну из сторон
6. Подтверждает транзакцию → маркет создан
7. Получает ссылку для шеринга маркета
8. Шарит в Telegram/Twitter → привлекает бетторов
9. Маркет закрывается → создатель получает 25% от fee (~1% от пула)

### Flow 3: Реферал приводит друга

1. Пользователь копирует реферальный код из вкладки "Referral"
2. Отправляет ссылку другу
3. Друг переходит по ссылке → реферальный код привязывается автоматически
4. Друг делает первую ставку (мин. $10 для активации рефкода)
5. Маркет закрывается → реферер получает долю от fee со ставки друга
6. Реферер видит статистику: сколько рефералов, заработок, в разделе "Referral"

## 7. Конкуренты

**Belief Market** (belief.market) — ближайший аналог. Та же модель: победитель = сторона с наибольшим объёмом, объёмы скрыты до закрытия. Отличие: централизованная платформа (Squarespace), нет on-chain transparency. 3-уровневые рефералы, фиатные валюты (PHP default). Таргет — Филиппины/SEA.

**Polymarket** — крупнейший prediction market, но основан на реальных исходах (выборы, спорт). Не belief-based. AMM модель, ордербук.

**Azuro** — протокол для prediction markets, ликвидность через пулы. Ориентирован на спортивные ставки. Возможный партнёр/интеграция для V1+.

Наше позиционирование: **on-chain belief-based betting с ZK privacy**. Единственная платформа, где слепота гарантирована криптографически, а не доверием к серверу.
